线性回归代码缺陷修复报告

一、概述

本次针对原始线性回归代码 linear_regression-tf2.0.py 存在的逻辑Bug、绘图错误进行定点修复，解决两大核心问题：偏置项b未参与梯度更新、图例标签与曲线数量不匹配，同时规范代码结构与可视化展示效果。

核心改进涵盖以下三个方面：

  1. 梯度计算完整性（同时对权重w和偏置b求梯度）
  2. 参数更新正确性（使用zip配对更新所有可训练参数）
  3. 可视化规范性（图例标签与绘制曲线严格对应）

二、原始代码存在问题

2.1 严重逻辑Bug：偏置项b从未更新

原关键代码

@tf.function
def train_one_step(model, xs, ys):
    with tf.GradientTape() as tape:
        y_preds = model(xs)
        loss = tf.keras.losses.MSE(ys, y_preds)
    # 仅对权重w求梯度，忽略偏置b
    grads = tape.gradient(loss, model.w)
    # 仅更新w，b始终为初始0值
    optimizer.apply_gradients([(grads, model.w)])
    return loss

问题说明

  1. 模型前向传播使用公式：$y = w\cdot x + b$，同时定义了可训练参数 w 和 b；
  2. 反向传播仅对权重 w 计算梯度、执行更新；
  3. 偏置项 b 全程固定初始值，没有参与任何训练迭代；
  4. 模型丧失截距拟合能力，只能拟合过原点直线，拟合精度大幅下降。

数学分析

线性回归的标准形式为：

y = w·x + b

其中： - w 是权重（斜率），控制直线的倾斜程度 - b 是偏置（截距），控制直线在y轴上的位置

梯度下降的更新规则为：

w_new = w_old - lr * ∂L/∂w
b_new = b_old - lr * ∂L/∂b

原始代码只执行了第一个更新，导致 b 始终为初始值0。

影响分析

设真实数据关系为 y = 2x + 3：

x 真实y 无偏置模型预测 (y=2x) 误差
1 5 2 3
5 13 10 3
10 23 20 3
  • 绝对误差恒为3（即偏置值）
  • 相对误差在小x值时巨大（x=1时为60%）
  • 模型无法捕捉数据的真实分布

2.2 可视化Bug：图例标签数量不匹配

原问题代码

plt.plot(o_x, o_y, "ro", markersize=3)      # 绘制训练数据点
plt.plot(o_x_test, y_test_preds, "k")        # 绘制预测曲线
plt.legend(["train", "test", "pred"])        # 3个标签，但只有2条曲线

问题说明

  • 绘图仅绘制训练集数据点、预测曲线2条线条
  • 但图例设置3个标签 ["train", "test", "pred"]
  • 出现多余无效图例，界面展示不规范
  • 读者会寻找不存在的"test"曲线，造成理解混淆

2.3 其他隐性问题

  • 损失函数未做均值归一化，数值波动偏大；
  • 代码梯度求解与参数更新写法不规范，扩展性差；
  • 学习率设置为0.1，对于Adam优化器来说偏高。

三、代码修复方案

3.1 修复偏置项梯度更新

修复后代码

@tf.function
def train_one_step(model, xs, ys):
    with tf.GradientTape() as tape:
        y_preds = model(xs)
        loss = tf.reduce_mean(tf.keras.losses.MSE(ys, y_preds))
    # 同时对w、b求梯度
    grads = tape.gradient(loss, [model.w, model.b])
    # 同时配对更新w和b
    optimizer.apply_gradients(zip(grads, [model.w, model.b]))
    return loss

修复要点

  1. 将求梯度对象改为列表 [model.w, model.b]，同时求解两个参数梯度；
  2. 使用 zip 将梯度与参数一一绑定，同步更新权重与偏置；
  3. 增加 tf.reduce_mean 对损失做均值处理，训练更平稳。

TensorFlow梯度计算机制

# 单参数梯度
grads = tape.gradient(loss, model.w)  # 返回单个梯度张量

# 多参数梯度
grads = tape.gradient(loss, [model.w, model.b])  # 返回梯度列表

# 参数更新
optimizer.apply_gradients(zip(grads, [model.w, model.b]))
# zip将梯度与参数配对：[(grad_w, w), (grad_b, b)]

3.2 修复图例标签不匹配

修复方式

原多余测试集标签删除，图例严格与绘制线条数量对应：

plt.plot(o_x, o_y, "ro", markersize=3)      # 绘制训练数据点
plt.plot(o_x_test, y_test_preds, "k")        # 绘制预测曲线
plt.legend(["train", "pred"])                # 2个标签，与曲线对应

四、改进后完整代码结构

4.1 模型定义

class LinearModel(Model):
    """线性回归模型，实现 y = w·x + b"""

    def __init__(self, ndim):
        super(LinearModel, self).__init__()
        # 权重参数：形状 [ndim, 1]
        self.w = tf.Variable(
            shape=[ndim, 1],
            initial_value=tf.random.uniform(
                [ndim, 1], minval=-0.1, maxval=0.1, dtype=tf.float32
            ),
            trainable=True,
            name="weight"
        )
        # 偏置参数：形状 [1]
        self.b = tf.Variable(
            initial_value=tf.zeros([1], dtype=tf.float32),
            trainable=True,
            name="bias"
        )

    @tf.function
    def call(self, x):
        y = tf.squeeze(tf.matmul(x, self.w), axis=1) + self.b
        return y

4.2 基函数变换

文件中定义了三种基函数，用于将原始输入映射到高维特征空间：

  1. 恒等基函数：直接返回输入，适用于线性关系
  2. 多项式基函数：将x映射为[x, x², x³, ...]，捕捉多项式关系
  3. 高斯基函数：将x映射为一组高斯分布特征，捕捉非线性关系
def gaussian_basis(x, feature_num=10):
    """高斯基函数：将输入x映射为一组高斯分布特征"""
    centers = np.linspace(0, 25, feature_num)  # 高斯中心点
    width = 1.0 * (centers[1] - centers[0])    # 高斯宽度
    x = np.expand_dims(x, axis=1)
    x = np.concatenate([x] * feature_num, axis=1)
    out = (x - centers) / width
    ret = np.exp(-0.5 * out ** 2)
    return ret

4.3 数据流

输入: x (原始数据)
  -> load_data(): 读取数据文件
  -> basis_func(): 基函数变换 (高斯/多项式/恒等)
  -> 添加偏置列: [1, phi1, phi2, ...]
  -> model(xs): 前向传播 y = w·x + b
  -> loss = MSE(ys, y_preds): 计算损失
  -> tape.gradient(): 计算梯度
  -> optimizer.apply_gradients(): 更新参数

五、原理补充：偏置项b的作用

5.1 线性回归标准模型

y = w x + b
  • $w$：权重，控制曲线斜率；
  • $b$：偏置/截距，控制曲线上下平移；

5.2 偏置项的几何意义

偏置值b 直线位置 适用场景
b = 0 通过原点 数据天然满足y(0)=0
b > 0 y轴截距为正 数据存在正向偏移
b < 0 y轴截距为负 数据存在负向偏移

5.3 偏置项的数学意义

在梯度下降优化中，偏置项的梯度计算为：

∂L/∂b = ∂L/∂y · ∂y/∂b = ∂L/∂y · 1

这意味着偏置项的梯度等于损失函数对预测值的梯度，是优化过程中不可或缺的一部分。

5.4 无偏置模型的局限性

若无b且不更新，模型强制曲线过原点，无法适配带偏移的真实数据分布：

  • 房价预测：即使面积为0，也可能有基础价格（土地价值）
  • 温度预测：即使时间为0，也可能有基础温度
  • 销量预测：即使广告投入为0，也可能有基础销量

七、总结

  1. 核心修复：补全偏置项b的梯度求解与参数更新，解决模型结构残缺问题；
  2. 可视化修复：精简图例标签，做到绘图与图例严格对应；
  3. 规范代码：损失增加均值处理、梯度更新写法标准化，便于后续扩展多项式、高斯基函数回归；
  4. 关键结论：线性回归中 权重w和偏置b必须同时参与训练，缺一不可。

可继续改进的方向

  • 学习率调度：添加学习率衰减策略，如指数衰减或余弦退火
  • 正则化：添加L1/L2正则化，防止过拟合
  • 早停机制：监控验证集损失，自动停止训练
  • 模型评估：添加R²、MAE等更多评估指标
  • 数据增强：对训练数据添加噪声，提升泛化能力
